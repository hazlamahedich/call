# Plan: Single-Provider AI Abstraction (Embeddings + LLM + Frontend Settings)

```

## Context

The app currently hardcodes **OpenAI** for embeddings (`text-embedding-3-small`, 1536 dims) and has **zero direct LLM usage** — all conversational AI is delegated to Vapi's platform. The PRP (Sections 4.4, Phase 4) plans features that require LLM calls: AI script generation, call summarization, objection handling, and campaign optimization. Neither an embedding provider abstraction nor an LLM provider abstraction exists.

```

**Goal**: Create a unified `AI_PROVIDER` setting that controls both embeddings and LLM. One provider for everything. Support OpenAI and Gemini. Include a frontend settings page for admins.

```

## Decisions Made
| Decision | Choice | Rationale |
|---|---|---|
| Provider granularity | **Single `AI_PROVIDER`** | User picks one provider; both embedding and LLM use it |
| Gemini embedding impl | **Native `google-genai` SDK** | Full task type support, multimodal future-proofing |
| Gemini embedding dims | **3072 (native best)** | Best quality; migration nulls existing vectors, requires re-ingest |
| Frontend settings page | **Yes, included** | Admin UI for provider/model/API key management |
| Vapi assistant model | **Deferred** | Out of scope; Vapi dashboard handles LLM config for calls |
| API key storage | **Encrypted in DB** (per-org) | Multi-tenant provider config |
```

## Phase 2: Embedding Provider Abstraction

### 2.1 Create Provider Interface

**New file: `apps/api/services/embedding/providers/base.py`**

```python
from abc import ABC, abstractmethod
from typing import List

class EmbeddingProvider(ABC):
    @abstractmethod
    async def embed(self, text: str) -> List[float]: ...

    @abstractmethod
    async def embed_batch(self, texts: List[str]) -> List[List[float]]: ...

    @property
    @abstractmethod
    def model_name(self) -> str: ...

    @property
    @abstractmethod
    def dimensions(self) -> int: ...
```

### 2.2 Implement Providers

**New file: `apps/api/services/embedding/providers/openai_provider.py`**

- Uses `AsyncOpenAI` (existing dependency)
  wraps `client.embeddings.create(model=..., input=...)`
- No new dependencies needed
- `task_type` param accepted but ignored (OpenAI has no equivalent)
  **New file: `apps/api/services/embedding/providers/gemini_provider.py`**
  Uses the native `google-genai` SDK:

```python
from google import genai
from google.genai import types

class GeminiEmbeddingProvider(EmbeddingProvider):
    def __init__(self, api_key: str, model: str = "gemini-embedding-001", dimensions: int = 3072):
        self._client = genai.Client(api_key=api_key)
        self._model = model
        self._dimensions = dimensions

    async def embed(self, text: str) -> List[float]:
        result = self._client.models.embed_content(
            model=self._model,
            contents=text,
            config=types.EmbedContentConfig(
                output_dimensionality=self._dimensions,
                task_type="RETRIEVAL_DOCUMENT",
            )
        )
        return result.embeddings[0].values

    async def embed_batch(self, texts: List[str]) -> List[List[float]]:
        result = self._client.models.embed_content(
            model=self._model,
            contents=texts,
            config=types.EmbedContentConfig(
                output_dimensionality=self._dimensions,
                task_type="RETRIEVAL_DOCUMENT",
            )
        )
        return [e.values for e in result.embeddings]
```

**New dependency**: `google-genai` pip package (add to `requirements.txt`)
**Task type mapping**: Use `RETRIEVAL_DOCUMENT` for ingestion, `RETRIEVAL_QUERY` for search. The provider exposes a `set_task_type()` method or or the service passes it through.

## Phase 3: LLM Provider Abstraction

### 3.1 Create Provider Interface

**New file: `apps/api/services/llm/providers/base.py`**

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import AsyncIterator, List, Optional

@dataclass
class LLMMessage:
    role: str        # "system" | "user" | "assistant"
    content: str

@dataclass
class LLMResponse:
    content: str
    model: str
    usage: dict      # {"prompt_tokens": int, "completion_tokens": int}

class LLMProvider(ABC):
    @abstractmethod
    async def complete(
        self,
        messages: List[LLMMessage],
        *,
        temperature: float = 0.7,
        max_tokens: int = 2048,
        **kwargs,
    ) -> LLMResponse: ...

    @abstractmethod
    async def stream(
        self,
        messages: List[LLMMessage],
        *,
        temperature: float = 0.7,
        max_tokens: int = 2048,
        **kwargs,
    ) -> AsyncIterator[str]: ...

    @property
    @abstractmethod
    def model_name(self) -> str: ...
```

### 3.2 Implement Providers

**New file: `apps/api/services/llm/providers/openai_provider.py`**

- Uses `AsyncOpenAI` (existing dependency)
- Wraps `client.chat.completions.create(model=..., messages=..., stream=...)`
- Maps `LLMMessage` -> OpenAI message format
  **New file: `apps/api/services/llm/providers/gemini_provider.py`**
  Uses native `google-genai` SDK (same package as embeddings):

```python
class GeminiLLMProvider(LLMProvider):
    def __init__(self, api_key: str, model: str = "gemini-2.0-flash"):
        self._client = genai.Client(api_key=api_key)
        self._model = model

    async def complete(self, messages, *, temperature=0.7, max_tokens=2048, **kw) -> LLMResponse:
        contents = self._map_messages(messages)
        config =types.GenerateContentConfig(
            temperature=temperature,
            max_output_tokens=max_tokens,
        )
        result = await self._client.aio.models.generate_content(
            model=self._model, contents=contents, config=config,
        )
        return LLMResponse(content=result.text, model=self._model, usage={...})
    async def stream(self, messages, **kw) -> AsyncIterator[str]:
        result = await self._client.aio.models.generate_content(
            model=self._model, contents=self._map_messages(messages), config=config,
        )
        async for chunk in result:
            yield chunk.text
```

No additional dependency — reuses `google-genai` from embeddings.

## Phase 5: Database Migration for Configurable Dimensions

### File: New migration `apps/api/migrations/versions/xxxx_configurable_embedding_dimensions.py`

When switching to Gemini (3072 dims):

```sql
-- 1. Drop the existing HNSW index
DROP INDEX IF EXISTS idx_knowledge_chunks_embedding_hnsw;

-- 2. Null out existing embeddings (incompatible with new dimensions)
UPDATE knowledge_chunks SET embedding = NULL, embedding_model = NULL WHERE embedding IS NOT NULL;
-- 3. Alter the vector column to new dimension
ALTER TABLE knowledge_chunks ALTER COLUMN embedding TYPE vector(3072);
-- 4. Recreate HNSW index
CREATE INDEX idx_knowledge_chunks_embedding_hnsw
ON knowledge_chunks USING hnsw (embedding vector_cosine_ops)
WITH (m =16, ef_construction=256)
WHERE soft_delete=false AND embedding IS NOT NULL;
-- 5. Mark affected docs for re-ingestion
UPDATE knowledge_base SET status='processing'
WHERE id IN (SELECT DISTINCT knowledge_base_id FROM knowledge_chunks WHERE embedding IS NULL);
```

**Important**: Switching to Gemini at 3072 dims **invalidates all existing embeddings**. Documents are marked back to `processing` status. User must re-ingest from UI.
trigger a background re-embedding.
